# MI300X（GFX942 / CDNA3）优化指南

同架构变体：**MI325X** 亦为 GFX942 / CDNA3，与 MI300X 共享相同 CU 数与 ISA；主要差异为 HBM 容量与介质（见下「变体说明」）。

## 核心参数

| 参数 | 值 |
|------|-----|
| 架构 | CDNA3 |
| LLVM / offload-arch | `gfx942` |
| 计算单元 (CU) | **304**（8 XCD × **38** active CU/XCD；每 XCD 共 **40** CU，**4** 个 disabled 用于良率；**4** ACEs/XCD，白皮书口径） |
| HBM | 192 GiB HBM3 |
| IOD | **4** 个（CDNA3；CDNA4 为 2，见 MI355X 文档） |
| Infinity Fabric（单链路） | **32 Gbps**/link（CDNA3 白皮书） |
| L2 聚合读带宽 | **34.4 TB/s**（aggregate read，白皮书） |
| Infinity Cache 带宽 | **17.2 TB/s**（白皮书） |
| Wavefront 大小 | 64 |
| LDS / CU | 64 KiB |
| L3 Cache（Infinity Cache） | 256 MiB |
| L2 Cache | 32 MiB（每 XCD 4 MiB，共 8 XCD） |
| L1 Vector Cache / CU | 32 KiB |
| L1 Scalar Cache | 16 KiB / 2 CUs |
| L1 Instruction Cache | 64 KiB / 2 CUs |
| VGPR File / CU | 512 KiB |
| SGPR File / CU | 12.5 KiB |
| Matrix Core 数量 | 1216（每 CU 4 个，304 × 4） |
| Matrix 峰值（官方标称） | FP16 **1307.4** TFLOPS；BF16 **1307.4** TFLOPS；FP8 **2614.9** TFLOPS；FP64 **163.4** TFLOPS |
| Max engine clock | 2100 MHz |
| 最大线程数 / CU | 2048（32 个 wavefront × 64） |

## Partition 与 NPS 模式（多 XCD / NUMA）

MI300X 类 **8 XCD** 设备支持多种 **partition** 与 **NPS（NUMA Per Socket）** 组合，影响可见 GPU 拓扑、隔离度与访存局部性：

| 模式 | 含义 | 典型用途 |
|------|------|----------|
| **SPX**（Single Partition eXperience） | **8 个 XCD** 作为 **单 GPU** 暴露 | **默认**；追求最大单卡算力与统一地址空间 |
| **DPX**（Dual Partition） | **2× 虚拟 GPU**，各 **4 XCD** | **推理 serving** 等需双租户或双实例隔离的场景 |
| **CPX**（Compute Partition eXperience） | **8× 独立 partition** | **极强隔离**（多 job / 多租户硬分） |

**CDNA4 补充**：**GFX950** 另增 **QPX**（**4** partitions × **2** XCD）等模式，且 **NPS** 选项与 **IOD=2** 布局不同于 CDNA3；详见 [`mi355x.md`](mi355x.md)。

**NPS（NPS1 / NPS2 / NPS4）**：控制 **内存交错（interleaving）** 策略与 socket 内 NUMA 行为；与 partition 组合会改变带宽与延迟特性。

- **CPX + NPS4**：资料称可通过 **局部化 HBM 访问** 获得约 **5–10%** 更高 **内存带宽**（相对部分默认/交错配置）；具体以平台 BIOS / 系统设置与实测为准。

调优 persistent kernel、多进程或多实例部署时，应在目标 **partition + NPS** 组合下重新 benchmark。

## 编译

```bash
# HIP
hipcc -O3 --offload-arch=gfx942 kernel.cpp

# Triton: 自动检测或由后端指定目标，与 gfx942 一致时需确认 ROCm 版本支持

# CK: 使用 CMake，设置 AMDGPU_TARGETS=gfx942
```

## 性能阈值参考

| 指标 | 良好 | 需改进 |
|------|------|--------|
| Occupancy | > 50% | < 25% |
| HBM 带宽利用率 | > 60%（内存受限场景） | < 30% |
| MFMA 利用率 | > 70%（计算受限场景） | < 40% |
| LDS Bank Conflict | < 5% | > 20% |
| 寄存器溢出 | 0 scratch waves | 任何 scratch |

## 硬件特有技巧

- **304 CU** 并行度高；极小 grid 可能吃不满设备，可考虑 persistent kernel 或合并 launch。
- **64-wide wavefront**：规约类 shuffle 步数与 32-wide 不同，勿直接套用 NVIDIA warp 假设。
- **缓存层级**：L3 为 256 MiB（Infinity Cache），L2 总容量为 32 MiB；tile 遍历顺序对 L2/L3 复用影响大，勿与旧文档中「L2 = 256 MB」混淆。
- **MFMA**：矩阵核与峰值以官方 TFLOPS 为准；调优时结合 `rocprof`/Nsight 类工具区分 memory-bound 与 compute-bound。

## 已知问题

- `hipcc` 省略 `-O3` 时可能静默回退到较慢的指令模式。
- Triton ROCm 后端对 `tl.inline_asm_elementwise` 等能力的支持因版本而异，需查发行说明。
- 与 NVIDIA 不同的 grid launch 限制 — 使用 `hipDeviceGetAttribute` 查询上限。
